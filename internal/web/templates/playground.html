A playground is an isolated copy of a database (or a pre-built sample) that lives for the duration of this session. Use it to safely experiment with queries, test schema changes, or learn FlashAccess without touching live data. {{if .PlaygroundDB}}Creating a new playground will replace the existing one.{{end}}

The selected database will be copied into the playground. Your live database is not modified.

What can I do with a playground?
⚡
Preview destructive queries

Run DELETE, UPDATE, or DROP statements here first. See exactly what changes before touching the real thing.

🎓
Learn SQL safely

The sample database has foreign keys, indexes, and realistic data. A perfect sandbox for practicing joins, aggregations, and schema changes.

🔬
Test schema migrations

Clone your production database and run ALTER TABLE or migration files here to validate before deploying.

🗑
Auto-cleanup

The playground database is automatically dropped when your session ends. No cleanup required.
