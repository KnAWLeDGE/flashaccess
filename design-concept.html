Temporary Root Accessfor MySQL. Secured.

FlashAccess creates time-limited, IP-restricted sessions for root-level MySQL management — without permanently exposing your database. Runs behind Nginx + Cloudflare with full TLS.

Install
curl -fsSL https://flashaccess.dev/install.sh | bash
What FlashAccess does
01
Session-Keyed Access
Every access window is tied to a unique cryptographic key. No key, no entry — for the web UI and SSH tunnel alike.
02
IP Restriction
Lock sessions to a specific IP address. Cloudflare-aware: reads CF-Connecting-IP headers to enforce the real client IP behind the proxy.
03
Time-Limited Durations
Set sessions from 1 to 24 hours. Auto-terminates on expiry — or immediately via the dashboard or CLI.
04
Web Dashboard
A full PHPMyAdmin-style management interface served only during an active session. Disappears when the session ends.
05
SSH Dev Commands
Developers with server access can use CLI commands within the same session to open root access for tools like Navicat.
06
Nginx + Cloudflare Ready
Works out of the box with Cloudflare Full (strict) TLS mode and Nginx reverse proxy. No special firewall rules needed.
How it works
1
Install on your Ubuntu server
Run the install script. FlashAccess installs as a systemd service, configures Nginx, and initialises its local SQLite state store. No cloud dependencies.
2
Connect your MySQL database
Point FlashAccess at your MySQL instance with a one-time flashaccess connect command. Credentials are stored encrypted on-disk.
3
Start a session
Run flashaccess session start. Choose your duration, allowed IP, and security options. A session key is generated.
4
Access the dashboard or open an SSH tunnel
Navigate to your domain — the management UI is live only during the session. Or use flashaccess tunnel for direct tool access (Navicat, TablePlus, etc.).
5
Session terminates automatically
Access closes on expiry, IP change, or manual kill. No residual open ports. Audit log written to disk.
