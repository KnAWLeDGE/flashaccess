FlashAccess Documentation

FlashAccess gives you a time-limited, IP-restricted web dashboard and MySQL credentials for a single session. When time runs out the temp user is dropped and the firewall rule is removed automatically.

Requirements & detection

FlashAccess requires the following on your Ubuntu 24.04 server. Use the detection script below to automatically check what is already present.

  • Ubuntu 24.04 (22.04 works; other Linux distros need manual UFW setup)
  • MySQL 8.0+ running locally — install guide ↓ optional setup
  • nginx — install guide ↓ optional setup
  • UFW enabled — setup guide ↓ optional setup
  • A domain behind Cloudflare with Full (Strict) TLS — walkthrough ↓ optional setup
  • A Cloudflare origin certificate — created in step 5 of the Cloudflare guide
Note: FlashAccess itself does not open any public ports. It listens on 127.0.0.1:7432 and is exposed only through nginx. UFW rules it manages are for direct MySQL TCP access used by the CLI activate path.

Server detection script

Run this on your Ubuntu server (as root or with sudo). It outputs JSON describing what is already installed. Paste the result into the field below to automatically check off requirements. 

bash <<'EOF'
#!/usr/bin/env bash
detect() {
  local key="$1"; shift
  if command -v "$1" &>/dev/null; then
    local v; v=$("$@" 2>&1 | head -1 | sed 's/"/\\"/g')
    printf '  "%s": "%s"' "$key" "$v"
  else
    printf '  "%s": null' "$key"
  fi
}
printf "{\n"
printf "  "; detect mysql    mysqladmin --version;  printf ",\n"
printf "  "; detect nginx    nginx -v;              printf ",\n"
printf "  "; detect php      php --version;         printf ",\n"
printf "  "; detect python3  python3 --version;     printf ",\n"
printf "  "; detect go       go version;            printf ",\n"
printf "  "; detect node     node --version;        printf ",\n"
printf "  "; detect redis    redis-cli --version;   printf ",\n"
printf "  "; detect docker   docker --version;      printf ",\n"
printf "  "; detect git      git --version;         printf ",\n"
printf "  "; detect composer composer --version;    printf ",\n"
printf "  "; detect flashaccess flashaccess version; printf ",\n"
ufw_status=$(ufw status 2>/dev/null | awk '/^Status:/{print $2}')
printf '  "ufw": "%s",\n' "${ufw_status:-unknown}"
printf '  "fa_config": %s,\n' "$([ -f /var/lib/flashaccess/config.enc ] && echo '"present"' || echo 'null')"
fa_svc=$(systemctl is-active flashaccess 2>/dev/null | tr -d '\n'); [ -z "$fa_svc" ] && fa_svc="not-found"
printf '  "fa_service": "%s"\n' "$fa_svc"
printf "}\n"
EOF

MySQL server optional — skip if MySQL is already installed

FlashAccess connects to an existing MySQL 8.0+ instance running on the same server. If MySQL is already installed, skip this section entirely.

Install MySQL Server on Ubuntu 24.04:

$ sudo apt update
$ sudo apt install -y mysql-server
$ sudo systemctl enable --now mysql

Run the security hardening wizard (recommended):

$ sudo mysql_secure_installation

The wizard will ask you to:

  • Set or validate the root password policy
  • Remove anonymous users
  • Disallow remote root login
  • Remove the test database

Verify MySQL is running and accessible:

$ sudo mysql -u root
mysql> SHOW DATABASES;
mysql> EXIT;

Allow socket authentication (recommended)

By default Ubuntu's MySQL uses socket authentication for root — no password needed when running as root. This is the most secure setup and is what FlashAccess supports out of the box (leave the MySQL password blank in flashaccess connect): 

$ sudo mysql -u root -e "SELECT user, plugin FROM mysql.user WHERE user='root';"
# Should show: plugin = auth_socket or caching_sha2_password

If you prefer password auth, set a root password. First open a MySQL shell:

$ sudo mysql

Then run inside MySQL:

mysql> ALTER USER 'root'@'localhost' IDENTIFIED WITH caching_sha2_password BY 'yourpassword';
mysql> FLUSH PRIVILEGES;
mysql> EXIT;

nginx optional — skip if nginx is already installed

FlashAccess listens on 127.0.0.1:7432 and must be reverse-proxied through nginx. If nginx is already installed, skip this section. 

$ sudo apt update
$ sudo apt install -y nginx
$ sudo systemctl enable --now nginx

Allow nginx through UFW:

$ sudo ufw allow 'Nginx Full'   # opens ports 80 and 443

Verify nginx is running:

$ curl -I http://localhost
# Should return: HTTP/1.1 200 OK
$ sudo nginx -t   # check config syntax

nginx configuration for FlashAccess is handled during installation — the installer writes a sample config to /etc/nginx/sites-available/flashaccess. The nginx + Cloudflare TLS deployment section covers enabling it.

UFW firewall optional — skip if UFW is already enabled

FlashAccess uses UFW to open and close MySQL TCP access per session. UFW must be installed and enabled. If ufw status already shows active, skip this section.

Important: Allow SSH before enabling UFW, or you will lock yourself out of the server. 
$ sudo apt install -y ufw
$ sudo ufw default deny incoming
$ sudo ufw default allow outgoing
$ sudo ufw allow ssh           # MUST do this first
$ sudo ufw allow 'Nginx Full'  # ports 80 + 443
$ sudo ufw enable
$ sudo ufw status verbose
Do not open port 3306 (MySQL) in UFW manually — FlashAccess manages that automatically per session, opening it only for the session's allowed IP and closing it again when the session ends.

Cloudflare + domain setup optional — skip if already configured

FlashAccess requires a domain proxied through Cloudflare with Full (Strict) TLS. This walkthrough covers a fresh setup from scratch. Skip any step already completed.

  1. 1

    Register a domain

    Buy a domain from any registrar. Cloudflare Registrar is cheapest (at-cost, no markup). For a database tool you typically want a subdomain like db.yourdomain.com. If you already have a domain, go to step 2.

  2. 2

    Add the domain to Cloudflare

    In Cloudflare dashboard → Add a domain:

    1. Enter your root domain (e.g. yourdomain.com) and choose Free plan
    2. Cloudflare scans existing DNS — review and confirm
    3. Copy the two Cloudflare nameservers shown
    4. Go to your registrar's DNS settings and replace the existing nameservers with the Cloudflare ones

    Propagation typically takes 5–60 minutes. Cloudflare will email you when it's active.

  3. 3

    Add an A record pointing to your server

    Cloudflare → your domain → DNS → Records → Add record:

    • Type: A
    • Name: db (creates db.yourdomain.com) or @ for the root
    • IPv4 address: your server's public IP
    • Proxy status: Proxied (orange cloud) — required
  4. 4

    Set SSL/TLS mode to Full (Strict)

    Cloudflare → your domain → SSL/TLS → Overview → select Full (strict).

    Do not use "Flexible" — it allows unencrypted HTTP between Cloudflare and your server and breaks FlashAccess IP enforcement.
  5. 5

    Create a Cloudflare origin certificate

    Cloudflare → your domain → SSL/TLS → Origin Server → Create Certificate:

    • Let Cloudflare generate the private key
    • Add hostnames: yourdomain.com and *.yourdomain.com
    • Choose 15-year validity, then click Create

    Copy the certificate and private key (shown once). On your server:

    $ sudo mkdir -p /etc/ssl/cloudflare
$ sudo nano /etc/ssl/cloudflare/origin.crt   # paste the certificate
$ sudo nano /etc/ssl/cloudflare/origin.key   # paste the private key
$ sudo chmod 600 /etc/ssl/cloudflare/origin.key
$ sudo chmod 644 /etc/ssl/cloudflare/origin.crt
  6. 6

    (Optional) Lock nginx to Cloudflare IPs only

    To block clients who hit your server IP directly (bypassing Cloudflare), add these allow/deny directives inside your nginx server { } block: 

    # Cloudflare IPv4 ranges (keep updated from cloudflare.com/ips-v4)
allow 103.21.244.0/22;  allow 103.22.200.0/22;  allow 103.31.4.0/22;
allow 104.16.0.0/13;    allow 104.24.0.0/14;    allow 108.162.192.0/18;
allow 131.0.72.0/22;    allow 141.101.64.0/18;  allow 162.158.0.0/15;
allow 172.64.0.0/13;    allow 173.245.48.0/20;  allow 188.114.96.0/20;
allow 190.93.240.0/20;  allow 197.234.240.0/22; allow 198.41.128.0/17;
# Cloudflare IPv6
allow 2400:cb00::/32; allow 2606:4700::/32; allow 2803:f800::/32;
allow 2405:b500::/32; allow 2405:8100::/32; allow 2a06:98c0::/29; allow 2c0f:f248::/32;
deny all;

Install FlashAccess

Once all prerequisites are in place (MySQL, nginx, UFW, domain + Cloudflare), run the one-liner installer as root on your Ubuntu server: 

$ curl -fsSL https://raw.githubusercontent.com/KnAWLeDGE/flashaccess/main/install.sh | bash

The installer will:

  1. Detect your architecture (amd64 / arm64) and download the correct binary to /usr/local/bin/flashaccess
  2. Create a flashaccess system user
  3. Create /var/lib/flashaccess/ (mode 0700) for encrypted config and session state
  4. Install the systemd unit at /etc/systemd/system/flashaccess.service
  5. Write a sample nginx config to /etc/nginx/sites-available/flashaccess (if nginx is installed)

Manual install

Download a binary directly from GitHub Releases:

$ VER=$(curl -fsSL https://api.github.com/repos/KnAWLeDGE/flashaccess/releases/latest | grep '"tag_name"' | head -1 | sed 's/.*"\(.*\)".*/\1/')
$ wget "https://github.com/KnAWLeDGE/flashaccess/releases/download/${VER}/flashaccess_${VER}_linux_amd64"
$ chmod +x "flashaccess_${VER}_linux_amd64"
$ sudo mv "flashaccess_${VER}_linux_amd64" /usr/local/bin/flashaccess

flashaccess connect

After installation, run the configuration wizard as root to set up your MySQL connection and admin password: 

$ sudo flashaccess connect

The wizard prompts for:

  • MySQL host — default 127.0.0.1
  • MySQL port — default 3306
  • MySQL socket — leave blank for TCP; enter e.g. /var/run/mysqld/mysqld.sock for socket auth
  • MySQL user — typically root
  • MySQL password — leave blank if using socket auth
  • Dashboard listen address — default 127.0.0.1:7432
  • Admin password — minimum 8 characters; protects the session-creation form

The wizard verifies the MySQL connection before saving. Config is encrypted with AES-256-GCM and stored at /var/lib/flashaccess/config.enc. The master key is at /var/lib/flashaccess/master.key — back this up.

Warning: If you lose master.key the config cannot be decrypted. Run flashaccess connect again to reconfigure from scratch.

Because flashaccess connect is typically run as root, the config files may end up owned by root instead of the flashaccess service user. If the service fails to start with a permission denied error, fix ownership and restart: 

$ sudo chown -R flashaccess:flashaccess /var/lib/flashaccess
$ sudo systemctl restart flashaccess

Starting the server

Start FlashAccess manually to verify it works:

$ sudo flashaccess serve
FlashAccess dashboard listening on http://127.0.0.1:7432

Then enable and start it as a managed service:

$ sudo systemctl enable --now flashaccess
$ sudo systemctl status flashaccess

With nginx configured (see nginx + Cloudflare TLS), your dashboard will be accessible at https://db.yourdomain.com.

Sessions

A session is a time-limited access grant. It consists of a cryptographic key, an allowed IP/CIDR, a duration, and a temporary MySQL user provisioned for that session only.

Creating a session via the web dashboard

  1. Visit https://db.yourdomain.com
  2. Enter your admin password
  3. Fill in the allowed IP or CIDR (e.g. 203.0.113.42 or 10.0.0.0/24), session duration, and options
  4. Submit — the session key is shown once. Copy it now.
  5. You'll be redirected to the key gate. Paste the key to enter the dashboard.
Key shown once: The raw key is never stored — only its Argon2id hash is persisted. If you lose the key, end the session and create a new one.

Session duration

Sessions accept Go duration strings: 30m, 2h, 12h, 24h. The maximum is 24 hours. The timer is shown in the dashboard topbar and colour-shifts to amber at 15 minutes and red at 5 minutes.

Ending a session

Sessions end automatically on expiry. To end early:

  • Click End Session in the dashboard topbar, or
  • Run flashaccess session end <id> from the server CLI

On end: the temporary MySQL user is dropped, the UFW rule is closed, and all web tokens are revoked.

Web dashboard

The web dashboard is accessible only from the session's allowed IP. It provides:

  • Database list — all databases visible to the admin connection
  • Table browser — engine, row count, size; Browse and Structure per table
  • Row browser — paginated table view with sortable columns (25 / 50 / 100 rows)
  • Structure view — column info (type, nullable, key, default, extra, comment) and indexes
  • SQL editor — run arbitrary queries; Ctrl+Enter submits
Root access: The dashboard connects as your configured admin user (typically root). All SQL editor queries run with that privilege. There is no query filtering or sandbox.

CLI activate (Navicat / TablePlus / mysql)

To use a GUI client or the mysql CLI, the activate command verifies your session key and prints temporary credentials: 

$ flashaccess session activate <session-id> <key> <your-ip>

systemd service

The installer creates /etc/systemd/system/flashaccess.service. Key properties:

  • Runs as the dedicated flashaccess system user
  • Restarts automatically on failure (RestartSec=5s)
  • Hardened: NoNewPrivileges, ProtectSystem=strict, PrivateTmp
  • Listen address controlled by FLASHACCESS_ADDR env var
$ sudo systemctl enable --now flashaccess
$ sudo systemctl status flashaccess
$ sudo journalctl -u flashaccess -f   # follow logs

nginx + Cloudflare TLS

FlashAccess must run behind nginx with Cloudflare Full (Strict) TLS. IP enforcement reads CF-Connecting-IP — without Cloudflare proxying this header, any client could spoof their IP. See Cloudflare + domain setup for the certificate walkthrough.

First, make sure your Cloudflare origin certificate files are in place (created during Cloudflare setup): 

/etc/ssl/cloudflare/origin.crt
/etc/ssl/cloudflare/origin.key

Copy the config below into /etc/nginx/sites-available/flashaccess, replacing db.yourdomain.com with your actual subdomain: 

$ sudo nano /etc/nginx/sites-available/flashaccess

Paste this complete config (replace db.yourdomain.com):

# Validate that the request is actually coming through Cloudflare.
# Without this, anyone who knows your server IP could spoof CF-Connecting-IP.
geo $cloudflare_ip {
    default          0;
    103.21.244.0/22  1;
    103.22.200.0/22  1;
    103.31.4.0/22    1;
    104.16.0.0/13    1;
    104.24.0.0/14    1;
    108.162.192.0/18 1;
    131.0.72.0/22    1;
    141.101.64.0/18  1;
    162.158.0.0/15   1;
    172.64.0.0/13    1;
    173.245.48.0/20  1;
    188.114.96.0/20  1;
    190.93.240.0/20  1;
    197.234.240.0/22 1;
    198.41.128.0/17  1;
    2400:cb00::/32   1;
    2606:4700::/32   1;
    2803:f800::/32   1;
    2405:b500::/32   1;
    2405:8100::/32   1;
    2a06:98c0::/29   1;
    2c0f:f248::/32   1;
}

server {
    listen 443 ssl;
    listen [::]:443 ssl;
    server_name db.yourdomain.com;

    # Cloudflare origin certificate (Full Strict mode)
    ssl_certificate     /etc/ssl/cloudflare/origin.crt;
    ssl_certificate_key /etc/ssl/cloudflare/origin.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # Security headers
    add_header X-Frame-Options        DENY;
    add_header X-Content-Type-Options nosniff;
    add_header Referrer-Policy        strict-origin-when-cross-origin;

    # Only pass CF-Connecting-IP when the request truly comes from Cloudflare.
    # If not from Cloudflare, the header is blanked so FlashAccess rejects it.
    set $real_cf_ip "";
    if ($cloudflare_ip = 1) {
        set $real_cf_ip $http_cf_connecting_ip;
    }

    location / {
        proxy_pass         http://127.0.0.1:7432;
        proxy_http_version 1.1;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   CF-Connecting-IP  $real_cf_ip;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto https;
        proxy_set_header   Upgrade           $http_upgrade;
        proxy_set_header   Connection        "upgrade";
        proxy_read_timeout 60s;
    }
}

# Redirect plain HTTP to HTTPS
server {
    listen 80;
    listen [::]:80;
    server_name db.yourdomain.com;
    return 301 https://$host$request_uri;
}

Then enable the site and reload nginx:

$ sudo ln -s /etc/nginx/sites-available/flashaccess /etc/nginx/sites-enabled/
$ sudo nginx -t
$ sudo systemctl reload nginx
Set Cloudflare SSL/TLS mode to Full (Strict) in your Cloudflare dashboard (SSL/TLS → Overview). Flexible or Full without Strict will allow Cloudflare to accept invalid certificates and weakens the security chain.

UFW rules

FlashAccess calls ufw to open and close MySQL TCP access (port 3306) for the session's allowed IP. UFW must be installed and enabled — see UFW setup. Do not manually open port 3306; FlashAccess manages it automatically.

To disable firewall management (dev machine / no UFW):

Environment=FLASHACCESS_NOFW=1   # add to the [Service] section of the systemd unit

Security model

  • AES-256-GCM — config encrypted at rest with a per-machine master key
  • Argon2id — session keys hashed before storage; raw key never persisted
  • bcrypt — admin password hashed (cost 12); 500ms artificial delay on failure
  • IP enforcement — checked on every request via CF-Connecting-IP
  • Brute-force lockout — 5 failed key attempts locks the session for 5 minutes
  • HttpOnly / Secure / SameSite=Strict cookies — web tokens are secure cookies
  • In-memory token store — tokens not persisted; restart requires re-auth
  • Temporary MySQL users — named fa_<hexid>; dropped on session end
Threat model: FlashAccess protects against opportunistic access and credential leakage — not against an attacker who has already compromised the server. The master key at /var/lib/flashaccess/master.key must be protected.

Environment variables

  • FLASHACCESS_ADDR — override the listen address (default 127.0.0.1:7432)
  • FLASHACCESS_NOFW — set to 1 to skip all UFW calls (dev / testing)

Building from source

Requires Go 1.23+.

$ git clone https://github.com/KnAWLeDGE/flashaccess.git && cd flashaccess
$ go build -o bin/flashaccess ./cmd/flashaccess
$ sudo cp bin/flashaccess /usr/local/bin/flashaccess

Cross-compile Linux binaries:

$ VER=v1.0.1   # replace with the version you are building
$ GOOS=linux GOARCH=amd64 CGO_ENABLED=0 go build -trimpath -ldflags "-s -w -X main.version=${VER}" -o dist/flashaccess_${VER}_linux_amd64 ./cmd/flashaccess
$ GOOS=linux GOARCH=arm64 CGO_ENABLED=0 go build -trimpath -ldflags "-s -w -X main.version=${VER}" -o dist/flashaccess_${VER}_linux_arm64 ./cmd/flashaccess

Releasing a new version

  1. Commit all changes and ensure the working tree is clean
  2. Tag the release: git tag v1.0.1 && git push origin v1.0.1 (increment as needed)
  3. Cross-compile both Linux binaries (see above)
  4. Create a GitHub Release on the tag; upload the dist/ binaries as release assets
  5. The install script resolves the latest tag automatically via the GitHub API
