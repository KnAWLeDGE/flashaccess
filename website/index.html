What it does

Database access that cleans up after itself.

Designed for developers who need temporary root access — for a migration, a debug session, or an emergency fix — without leaving permanent credentials behind.

Session-scoped keys
Each session generates a one-time 256-bit key (Argon2id hashed at rest). Shown once. Expires with the session.
IP / CIDR locking
Access is restricted to a specific IP or CIDR block at the application layer. Works transparently behind Cloudflare.
Auto-expiry
Sessions expire automatically. The temporary database user is dropped and the firewall rule is closed the moment a session ends.
Web dashboard
Browse tables, inspect structure, and run queries — a self-contained MySQL management UI that lives inside the session.
CLI + Navicat support
flashaccess session activate outputs credentials for use with any MySQL client — Navicat, TablePlus, mysql CLI.
Single binary
One statically compiled Go binary. No runtime dependencies. The dashboard, CLI, and web server are all inside it.

How it works

Three commands, then you're in.

01
Install and configure
Run the one-liner installer, then configure your MySQL connection and set an admin password with the interactive wizard.
flashaccess connect
02
Start the dashboard
Enable and start the systemd service. FlashAccess listens on 127.0.0.1:7432 and is reverse-proxied through nginx.
systemctl enable --now flashaccess
03
Create a session
Visit your domain. Enter your admin password and the allowed IP. You'll receive a session key — valid for up to 24 hours, scoped to that IP only.
https://db.yourdomain.com → Session Setup
04
Access expires automatically
When time runs out (or you click End Session), the temp MySQL user is dropped and the UFW rule is removed. Nothing lingers.

Security model

Designed to be auditable.

FlashAccess does not add complexity to your database — it adds structure to temporary access.

AES-256-GCM configCredentials stored encrypted with a per-machine master key.
Argon2id key hashingSession keys are never stored in plaintext — only the Argon2id hash.
bcrypt admin passwordThe setup UI is gated by a bcrypt-hashed password with artificial delay on failure.
IP enforcement at app layerWorks correctly behind Cloudflare — reads CF-Connecting-IP, not RemoteAddr.
Temporary MySQL usersEach session creates a scoped user that is dropped on expiry — no permanent credentials.
UFW firewall rulesPort-level rules are opened and closed automatically via UFW for each session.
